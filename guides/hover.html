Hover Events

Hover vs. Mouse

Each plottable has two sets of 'hover' related methods: 


            .hover_in()
            .hover()
            .hover_out()

and


            .mouse_in()
            .mouse_move()
            .mouse_out()

This may seem redundant, and while the two are closely related, they serve very different purposes. The .hover() set of methods are the getter/setter methods. They are used to get or set the function to be called during each event. So passing a function to .hover_in() sets the function that will be called when the mouse enters the hover area. .hover() sets the function that is called when the mouse is moved while in the hover area, and .hover_out() sets the function that is called when the mouse leaves the hover area.

On the other hand, the .mouse_() set of methods are the actual event handlers. When the mouse enters the hover area, the .mouse_in() method is called and is responsible for calling the function that was set using .hover_in(). Similarly, the .mouse_move() method is called when the mouse moves, and is responsible for calling the function set by .hover(), and the .mouse_out() method is called when the mouse leaves the hover area, and is responsible for calling the function set by .hover_out().

It's important to keep the getter/setter methods separate from the event handlers because we don't want the event firing when using the getter. Additionally, as a result of keeping them separated, the .hover() set of methods are chainable, and should be used in the context of building a chart, while the .mouse() set of methods are not chainable, and should be used in the context of an event firing.
